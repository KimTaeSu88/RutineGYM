# RutineGym 개발 계획 (Plan)

## 목표
난이도/시간/타겟(3단계) 선택을 기반으로 “오늘의 코어 루틴”을 생성하고, 진행(타이머)과 기록(저장)을 제공하는 Python 기반 앱을 만든다.

## 범위(1차 MVP)
- 3단계 입력 UI: 난이도(초/중/고), 시간(10/20/30분), 타겟(전체/아랫뱃살/옆구리)
- 조용한 모드(기본 ON), 장비(푸시업 보드) 옵션
- 루틴 생성 결과(세트 수/휴식/동작 리스트 + 폼 팁/주의사항)
- 각 운동 항목 클릭/버튼으로 자세 참고 링크(유튜브/웹) 열기
- 루틴 저장: 히스토리/즐겨찾기(로컬 SQLite)

## 비범위(초기 제외)
- 개인화 추천(체중/경험/통증 상세 프로파일 기반), AI 코칭
- 음성 안내/자동 반복 카운트
- 클라우드 동기화/로그인

## 핵심 설계 결정
- 플랫폼: Python (1차는 Streamlit UI)
- 데이터 저장: SQLite (로컬)
- 루틴 생성: “동작 카탈로그 + 규칙 기반 필터링/샘플링”으로 시작하고, 추후 점수 기반 추천으로 확장

## 데이터 모델(초안)
- Exercise
  - id, name
  - url(자세 참고 링크: 유튜브/웹)
  - targets(전체/아랫뱃살/옆구리)
  - rep_type(reps/seconds), default_amount(난이도별 가중치로 계산)
  - noise_level(quiet/medium/loud), min_difficulty
  - form_tip, caution
- Routine
  - difficulty, minutes, target
  - items(Exercise + amount), set_count, rest_seconds
- Persistence
  - history: created_at, title, payload_json
  - favorites: created_at, title, payload_json

## 루틴 생성 규칙(초안)
- 휴식: 초급 60s / 중급 45s / 고급 30s
- 시간에 따른 구성:
  - 10분: 3개 동작 중심, 3~4세트
  - 20분: 4개 동작 중심, 3~4세트(저충격 유산소 섞기 가능)
  - 30분: 5개 동작 중심, 4~5세트(푸시업 보드 옵션 반영)
- 타겟 반영:
  - 전체 코어: 정적+동적 혼합
  - 아랫뱃살: 하복부 비중↑ + 허리 보호 동작 포함
  - 옆구리: 회전/측면 안정 비중↑
- 조용한 모드:
  - loud 동작 제외(기본)
  - 필요 시 medium도 제한하는 “초조용”은 추후 옵션

## UI/화면 계획(초안)
1) Builder 화면
- 3단계 선택 + 옵션 토글
- “루틴 생성” 버튼
2) Result 화면(같은 페이지 섹션)
- 루틴 카드(세트/휴식/동작)
- 동작별 “자세 영상/가이드 열기” 링크 버튼
- “히스토리 저장”, “즐겨찾기 저장”
3) Progress(타이머) 화면 (2차)
- 동작별 타이머/휴식 타이머, 일시정지/종료

## 리스크/검증 포인트
- “분 단위 시간 선택(10/20/30)”이 실제 사용에 충분한지(15분 등 커스텀 요구)
- 동작별 볼륨(회/초) 규칙이 너무 빡세거나 느슨하지 않은지(체감 난이도)
- 조용한 모드에서 20~30분 루틴이 지루하지 않게 구성 가능한지(동작 다양성)
